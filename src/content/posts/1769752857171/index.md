# 什么是BAC

bilibili-API-collect

哔哩哔哩 - API 收集整理

是Github BIliBiliApi 的重要文档，里面批露了BiliBili接口的一些调用方法。

如果你有做过BiliBIli的项目，那么对于此文档应该有所知道。

# 为什么

根据原仓库作者[所说](https://github.com/SocialSisterYi/bilibili-API-collect)：

>2026年1月28日，本仓库维护者收到B站委托的律师事务所发律师函警告邮件，指控本仓库中的项目存在“通过技术手段对哔哩哔哩平台非公开的API接口及其调用逻辑、参数结构、访问控制及安全认证机制进行系统性收集、整理，并以技术文档、代码示例等形式向不特定公众传播”的侵权行为。
>即日起停止维护并删除相关文档及源代码。

# 看法

对于一些国内平台的第三方开发，其实一直似乎处于不友好的阶段。

例如 [NeteaseCloudMusicApi](https://github.com/Binaryify/NeteaseCloudMusicApi)。

在某天我在写一个桌面端监听网易云实时播放内容时，我了解到Spotify提供了[开放Api](https://developer.spotify.com/)。

所以确实很难评价国内的大厂，因为我一方面也理解开发Api并不会带来多大的收益，还有可能提升运营成本，总之，RIP BAC.
